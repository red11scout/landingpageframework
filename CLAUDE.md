## SECTION 1: PROJECT IDENTITY & QUALITY STANDARDS

### 1.1 What You Are Building

You are building a production-grade, enterprise-class application that permanently replaces PowerPoint and static slide decks for BlueAlly Technology Solutions. This is not a presentation tool that mimics slides. It is a **living knowledge graph** â€” an interconnected network of content nodes that a presenter navigates dynamically based on audience needs, powered by three AI agents that continuously improve the structure and quality of the knowledge base.

The platform must meet the following non-negotiable standards:

**Quality Benchmark:** Top 1% of enterprise web applications. Every component must be visually stunning, functionally perfect, and brand-consistent. If a component looks like it was generated by a basic AI tool, rewrite it. The standard is what you would expect from a Stripe, Linear, or Vercel product page â€” clean, purposeful, premium.

**No Watermarks or External Branding:** Do not include "Made with AI," "Powered by Vercel," or any third-party branding visible to end users. The only brand on this platform is BlueAlly.

**No Login for Presenters:** General users and presenters access the platform without authentication. Authentication is required only for the Admin Console (`/admin` routes).

**Mobile-First Responsiveness:** Every page and component must be fully responsive. Test at 375px (iPhone SE), 768px (iPad), 1280px (laptop), and 1920px (desktop). The mobile experience must be excellent, not merely functional.

**Day/Night Mode:** Full light and dark mode support is mandatory on every page and component. Use `next-themes` with the `class` strategy.

**Logos Over Text:** Use the BlueAlly logo image in the navigation header, not text. Use icons alongside text labels throughout the UI.

### 1.2 Core Architectural Principles

The following five principles govern every architectural decision:

**Graph-First, Not Slide-First.** Content is stored as atomic nodes connected by typed edges. The presentation view is a traversal of this graph. There are no "slides" in the traditional sense â€” only nodes rendered in different display modes.

**AI as Collaborator, Not Replacement.** AI agents propose changes to the knowledge graph; humans approve them. Every AI-generated change is logged, reviewable, and reversible. The AI never silently modifies content.

**Offline-First, Online-Enhanced.** The application must function completely without internet connectivity. When online, it gains AI capabilities, real-time sync, and collaborative features. When offline, the full knowledge graph remains accessible.

**Data-Driven Customization.** Client-specific content is never manually formatted. It is always generated from a validated JSON schema, ensuring consistency and eliminating formatting errors.

**Brand as Code.** BlueAlly's visual identity is encoded in the Tailwind design token system and cannot drift. Every component, chart, and layout follows the same rules automatically.

### 6.3 Design Rules (Enforced)

These rules apply to every component without exception:

**Charts:** All Recharts components must use `ba-blue` (`#1A5DC8`) as the primary series color, `ba-sky` (`#4A9FE0`) as secondary, `ba-success` (`#38A169`) for positive trends, and `ba-error` (`#E53E3E`) for negative trends. Never use Recharts default colors.

**Buttons:** Primary buttons use `bg-ba-blue text-white hover:bg-ba-navy`. Secondary buttons use `border border-ba-blue text-ba-blue hover:bg-ba-light`. Destructive buttons use `bg-ba-error text-white`.

**Cards:** Use `bg-white dark:bg-ba-slate rounded-xl shadow-ba-md border border-ba-silver dark:border-ba-slate`.

**Node Cards on Graph:** Use `shadow-node` in default state, `shadow-node-active` when selected or hovered. Apply `animate-glow-pulse` to the currently active node during presentation mode.

**Typography:** Page titles use `text-display-md font-bold text-ba-navy dark:text-white`. Section headers use `text-xl font-semibold`. Body text uses `text-base text-ba-slate dark:text-ba-silver`.

**Spacing:** Use 4px grid (Tailwind default). Sections are separated by `py-16 md:py-24`. Cards have `p-6` internal padding.

**Loading States:** Use the `LoadingSpinner` component (a rotating BlueAlly flywheel SVG) for all async operations. Never use a plain spinner or skeleton without the BlueAlly brand.

```typescript
          where: eq(users.email, session.user.email),
        });
        if (user) {
          session.user.role = user.role;
        }
      }
      return session;
    },
  },

  },
});
```

```typescript
  const page = parseInt(searchParams.get('page') ?? '1');
  const limit = parseInt(searchParams.get('limit') ?? '50');
  const type = searchParams.get('type');
  const tag = searchParams.get('tag');
  const offset = (page - 1) * limit;

  let query = db.select().from(nodes).where(eq(nodes.is_canonical, true));
  if (type) query = query.where(eq(nodes.type, type as any));
  if (tag) query = query.where(arrayContains(nodes.tags, [tag]));

  const results = await query.limit(limit).offset(offset);
  return NextResponse.json({ nodes: results, page, limit });
}

export async function POST(req: NextRequest) {

  const parsed = CreateNodeSchema.safeParse(body);

  }

  // Generate embedding for semantic search
  const embeddingText = `${parsed.data.title} ${parsed.data.summary ?? ''} ${parsed.data.tags.join(' ')}`;
  const embedding = await generateEmbedding(embeddingText);
```

### 11.2 GraphRAG Retrieval (`src/lib/graphrag/retrieval.ts`)

```typescript
import { db } from '@/lib/db';
import { nodes, edges } from '@/lib/db/schema';
import { generateEmbedding } from './embed';
import { sql, inArray } from 'drizzle-orm';

export interface GraphRAGResult {
  seedNodes: any[];
  expandedNodes: any[];
  relevantEdges: any[];
  contextText: string;
}

export async function graphRAGRetrieve(query: string, topK = 5): Promise<GraphRAGResult> {
  // Step 1: Embed the query
  const embedding = await generateEmbedding(query);
  const embeddingStr = `[${embedding.join(',')}]`;

  // Step 2: Find top-K semantically similar nodes
  const seedResults = await db.execute(sql`
    SELECT id, type, title, summary, tags, industries,
           1 - (embedding <=> ${embeddingStr}::vector) AS similarity
    FROM nodes
    WHERE is_canonical = true AND embedding IS NOT NULL
    ORDER BY embedding <=> ${embeddingStr}::vector
    LIMIT ${topK}
  `);
  const seedNodes = seedResults.rows as any[];
  const seedIds = seedNodes.map((n) => n.id);

  // Step 3: Expand the subgraph (1-hop neighbors)
  const neighborEdges = await db.select().from(edges)
    .where(inArray(edges.source_node_id, seedIds));

  const neighborIds = [...new Set(neighborEdges.map((e) => e.target_node_id!))];
  const expandedNodes = neighborIds.length > 0
    ? await db.select({ id: nodes.id, type: nodes.type, title: nodes.title, summary: nodes.summary })
        .from(nodes).where(inArray(nodes.id, neighborIds))
    : [];

  // Step 4: Build context text for LLM
  const contextText = [
    '## Directly Relevant Knowledge Nodes:',
    ...seedNodes.map((n) => `- [${n.type.toUpperCase()}] ${n.title}: ${n.summary ?? 'No summary.'}`),
    '',
    '## Related Nodes (Connected via Knowledge Graph):',
    ...expandedNodes.map((n) => `- [${n.type.toUpperCase()}] ${n.title}: ${n.summary ?? 'No summary.'}`),
  ].join('\n');

  return { seedNodes, expandedNodes, relevantEdges: neighborEdges, contextText };
}
```

### 12.1 Librarian Agent (`src/lib/agents/librarian.ts`)

The Librarian Agent ingests raw content and builds the knowledge graph.

```typescript
import { generateObject } from 'ai';
import { anthropic } from '@ai-sdk/anthropic';
import { z } from 'zod';
import { db } from '@/lib/db';
import { nodes, edges, ai_proposals } from '@/lib/db/schema';
import { generateEmbedding } from '@/lib/graphrag/embed';

const ExtractionSchema = z.object({
  nodes: z.array(z.object({
    type: z.enum(['concept', 'case_study', 'solution', 'metric', 'customer']),
    title: z.string(),
    summary: z.string().max(300),
    tags: z.array(z.string()),
    industries: z.array(z.string()),
    content_markdown: z.string(),
  })),
  relationships: z.array(z.object({
    source_title: z.string(),
    target_title: z.string(),
    relationship_type: z.enum(['related_to', 'prerequisite', 'supports', 'customer_of', 'uses_technology', 'industry_match']),
    weight: z.number().min(0).max(1),
    reasoning: z.string(),
  })),
});

export async function runLibrarianAgent(documentText: string, sourceFile: string) {
  // Step 1: Extract nodes and relationships from document
  const { object } = await generateObject({
    model: anthropic('claude-3-5-sonnet-20241022'),
    schema: ExtractionSchema,
    prompt: `You are the BlueAlly Knowledge Librarian. Analyze this document and extract:
1. Atomic knowledge nodes (concepts, solutions, case studies, metrics, customers).
2. Relationships between those nodes.

Be thorough. Extract every distinct concept. Keep summaries factual and under 300 characters.

Document:
${documentText.slice(0, 50000)}`,
  });

  // Step 2: For each extracted node, check for near-duplicates
  for (const extractedNode of object.nodes) {
    const searchResult = await graphRAGRetrieve(extractedNode.title, 3);
    const topMatch = searchResult.seedNodes[0];

    if (topMatch && topMatch.similarity > 0.92) {
      // Propose a merge instead of creating a duplicate
      await db.insert(ai_proposals).values({
        agent_type: 'librarian',
        proposal_type: 'merge_nodes',
        proposal_data: {
          existing_node_id: topMatch.id,
          existing_title: topMatch.title,
          new_content: extractedNode,
          source_file: sourceFile,
        },
        confidence: topMatch.similarity,
        reasoning: `New content "${extractedNode.title}" is ${(topMatch.similarity * 100).toFixed(1)}% similar to existing node "${topMatch.title}". Proposing merge to maintain single source of truth.`,
      });
    } else {
      // Create new node
      const embedding = await generateEmbedding(`${extractedNode.title} ${extractedNode.summary} ${extractedNode.tags.join(' ')}`);
      await db.insert(nodes).values({
        type: extractedNode.type,
        title: extractedNode.title,
        summary: extractedNode.summary,
        content: { type: 'markdown', markdown: extractedNode.content_markdown },
        tags: extractedNode.tags,
        industries: extractedNode.industries,
        source_file: sourceFile,
        embedding,
      });
    }
  }

  // Step 3: Propose new edges
  for (const rel of object.relationships) {
    await db.insert(ai_proposals).values({
      agent_type: 'librarian',
      proposal_type: 'new_edge',
      proposal_data: {
        source_title: rel.source_title,
        target_title: rel.target_title,
        relationship_type: rel.relationship_type,
        weight: rel.weight,
      },
      confidence: rel.weight,
      reasoning: rel.reasoning,
    });
  }

  return { nodesProcessed: object.nodes.length, relationshipsProposed: object.relationships.length };
}
```

### 12.2 Director Agent (`src/lib/agents/director.ts`)

The Director Agent runs weekly and analyzes presentation telemetry to propose structural improvements.

```typescript
import { generateObject } from 'ai';
import { anthropic } from '@ai-sdk/anthropic';
import { z } from 'zod';
import { db } from '@/lib/db';
import { presentation_telemetry, nodes, edges, ai_proposals } from '@/lib/db/schema';
import { sql, gte, and, isNull } from 'drizzle-orm';

const DirectorProposalSchema = z.object({
  proposals: z.array(z.object({
    type: z.enum(['new_edge', 'restructure', 'archive', 'content_refresh']),
    title: z.string(),
    description: z.string(),
    confidence: z.number().min(0).max(1),
    data: z.record(z.unknown()),
  })),
  health_summary: z.string(),
});

export async function runDirectorAgent() {
  // Step 1: Gather telemetry from the past 7 days
  const sevenDaysAgo = new Date(Date.now() - 7 * 24 * 60 * 60 * 1000);

  const recentTelemetry = await db.select().from(presentation_telemetry)
    .where(gte(presentation_telemetry.created_at, sevenDaysAgo));

  // Step 2: Find sequential node pairs not connected by edges
  const sequentialPairs = new Map<string, number>();
  for (let i = 0; i < recentTelemetry.length - 1; i++) {
    const curr = recentTelemetry[i];
    const next = recentTelemetry[i + 1];
    if (curr.session_id === next.session_id && curr.node_id !== next.node_id) {
      const key = `${curr.node_id}:${next.node_id}`;
      sequentialPairs.set(key, (sequentialPairs.get(key) ?? 0) + 1);
    }
  }

  // Step 3: Find nodes with zero views
  const allNodes = await db.select({ id: nodes.id, title: nodes.title, view_count: nodes.view_count, created_at: nodes.created_at }).from(nodes);
  const orphanedNodes = allNodes.filter((n) => n.view_count === 0);

  // Step 4: Ask Claude to generate proposals
  const telemetrySummary = JSON.stringify({
    totalEvents: recentTelemetry.length,
    topSequentialPairs: Array.from(sequentialPairs.entries())
      .sort((a, b) => b[1] - a[1]).slice(0, 20)
      .map(([key, count]) => ({ pair: key, count })),
    orphanedNodes: orphanedNodes.slice(0, 20).map((n) => ({ id: n.id, title: n.title })),
  });

  const { object } = await generateObject({
    model: anthropic('claude-3-5-sonnet-20241022'),
    schema: DirectorProposalSchema,
    prompt: `You are the BlueAlly Knowledge Director. Analyze this weekly telemetry and propose structural improvements to the knowledge graph.

Telemetry Summary:
${telemetrySummary}

Generate proposals to:
1. Add edges between nodes that are frequently navigated together but not connected.
2. Flag nodes that have never been viewed for archival or content refresh.
3. Identify structural improvements to make the graph more navigable.

Be specific and actionable.`,
  });

  // Step 5: Save proposals to database
  for (const proposal of object.proposals) {
    await db.insert(ai_proposals).values({
      agent_type: 'director',
      proposal_type: proposal.type,
      proposal_data: proposal.data,
      confidence: proposal.confidence,
      reasoning: proposal.description,
    });
  }

  return { proposalsGenerated: object.proposals.length, healthSummary: object.health_summary };
}
```

```typescript
  }, [nodeData, currentNode, setCurrentNode]);

  // Record telemetry when leaving a node
  const recordTelemetry = useCallback(async (nodeId: string, prevNodeId?: string) => {
    const dwellMs = Date.now() - enterTime;
    await fetch('/api/telemetry', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        events: [{
          session_id: sessionId,
          node_id: nodeId,
          previous_node_id: prevNodeId,
          dwell_time_ms: dwellMs,
          navigation_type: 'click',
        }],
      }),
    });
    setEnterTime(Date.now());
  }, [sessionId, enterTime]);

  if (!currentNode) return <div className="flex items-center justify-center h-screen bg-ba-navy"><LoadingSpinner /></div>;

  return (
    <div className="relative w-screen h-screen bg-ba-navy overflow-hidden">
      {/* Main content */}
      <AnimatePresence mode="wait">
        <motion.div
          key={currentNode.id}
          initial={{ opacity: 0, x: 60 }}
          animate={{ opacity: 1, x: 0 }}
          exit={{ opacity: 0, x: -60 }}
          transition={{ duration: 0.3, ease: 'easeInOut' }}
          className="w-full h-full"
        >
          <NodeFullscreen node={currentNode} />
        </motion.div>
      </AnimatePresence>

      {/* Path breadcrumb â€” top bar */}

          onNavigate={(node) => {
            recordTelemetry(node.id, currentNode.id);
            addToPath(currentNode);
            setCurrentNode(node);
          }}
        />
      </div>

      {/* AI Assistant toggle â€” floating button */}
      <button
        onClick={() => setShowAssistant(!showAssistant)}
```

### 16.3 Sync Manager (`src/lib/offline/sync.ts`)

```typescript
import { offlineDb } from './db';

export async function syncToServer() {
  const queue = await offlineDb.sync_queue.toArray();
  if (queue.length === 0) return;

  for (const item of queue) {
    try {
      await fetch(item.endpoint, {
        method: item.method,
        headers: { 'Content-Type': 'application/json' },
        body: item.body,
      });
      await offlineDb.sync_queue.delete(item.id!);
    } catch {
      await offlineDb.sync_queue.update(item.id!, { retries: item.retries + 1 });
    }
  }
}

export async function queueForSync(endpoint: string, method: string, body: unknown) {
  await offlineDb.sync_queue.add({
    endpoint,
    method,
    body: JSON.stringify(body),
    created_at: new Date(),
    retries: 0,
  });
}

// Initialize sync on app load
if (typeof window !== 'undefined') {
  window.addEventListener('online', syncToServer);
}
```

```typescript
'use client';

import { useState, useEffect } from 'react';
import { WifiOff } from 'lucide-react';
import { motion, AnimatePresence } from 'framer-motion';

export function OfflineIndicator() {
  const [isOffline, setIsOffline] = useState(false);

  useEffect(() => {
    setIsOffline(!navigator.onLine);
    const handleOnline = () => setIsOffline(false);
    const handleOffline = () => setIsOffline(true);
    window.addEventListener('online', handleOnline);
    window.addEventListener('offline', handleOffline);
    return () => { window.removeEventListener('online', handleOnline); window.removeEventListener('offline', handleOffline); };
  }, []);

  return (
    <AnimatePresence>
      {isOffline && (
        <motion.div
          initial={{ y: -40, opacity: 0 }}
          animate={{ y: 0, opacity: 1 }}
          exit={{ y: -40, opacity: 0 }}
```

### 19.4 Zustand Presentation Store (`src/stores/presentationStore.ts`)

```typescript
import { create } from 'zustand';
import type { Node } from '@/lib/db/schema';

type NavigationMode = 'free' | 'guided' | 'curated';

interface PresentationStore {
  currentNode: Node | null;
  path: Node[];
  navigationMode: NavigationMode;
  isFullscreen: boolean;
  setCurrentNode: (node: Node) => void;
  addToPath: (node: Node) => void;
  goBack: () => void;
  setNavigationMode: (mode: NavigationMode) => void;
  setIsFullscreen: (value: boolean) => void;
  clearPath: () => void;
}

export const usePresentationStore = create<PresentationStore>((set, get) => ({
  currentNode: null,
  path: [],
  navigationMode: 'free',
  isFullscreen: false,
  setCurrentNode: (node) => set({ currentNode: node }),
  addToPath: (node) => set((state) => ({ path: [...state.path, node] })),
  goBack: () => set((state) => {
    const newPath = [...state.path];
    const prev = newPath.pop();
    return { path: newPath, currentNode: prev ?? state.currentNode };
  }),
  setNavigationMode: (mode) => set({ navigationMode: mode }),
  setIsFullscreen: (value) => set({ isFullscreen: value }),
  clearPath: () => set({ path: [], currentNode: null }),
}));
```

```typescript
import { QueryClient, QueryClientProvider } from '@tanstack/react-query';
import { useState } from 'react';

export function QueryProvider({ children }: { children: React.ReactNode }) {
  const [queryClient] = useState(() => new QueryClient({
    defaultOptions: {
      queries: {
        staleTime: 5 * 60 * 1000,   // 5 minutes
        gcTime: 30 * 60 * 1000,     // 30 minutes
        retry: (failureCount, error: any) => {
          // Don't retry on 4xx errors
          if (error?.status >= 400 && error?.status < 500) return false;
          return failureCount < 2;
        },
        networkMode: 'offlineFirst', // Use cached data when offline
      },
    },
  }));

  return <QueryClientProvider client={queryClient}>{children}</QueryClientProvider>;
}
```
