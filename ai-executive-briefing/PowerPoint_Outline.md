### Slide 14: Wave 1 — Symbolic AI (1950s–1980s)
**Headline:** Wave 1: Teaching Machines to Think in Rules
**Visual:** Timeline with key milestones:
- 1950: Turing's "Can machines think?"
- 1956: Dartmouth Conference coins "Artificial Intelligence"
- 1966: ELIZA chatbot at MIT
- 1972: MYCIN expert system
- 1974: First AI Winter
- 1980: Expert systems boom
- 1987: Second AI Winter
**Key Insight:** "Intelligence cannot be fully captured in hand-written rules."
**Speaker Notes:** The idea was elegant. Encode human knowledge as logical rules. If X and Y, then Z. For three decades, researchers built expert systems — a doctor's diagnostic logic, a chess player's strategy. Each rule written by hand. The problem was that the real world is messy. Rules break. Edge cases multiply. Two AI winters followed when hype exceeded reality.

### Slide 15: Wave 2 — Machine Learning (1990s–2010s)
**Headline:** Wave 2: Letting Data Do the Teaching
**Visual:** Timeline with key milestones:
- 1997: Deep Blue defeats Kasparov
- 2006: Hinton's deep learning revival
- 2011: IBM Watson wins Jeopardy!
- 2012: AlexNet wins ImageNet
**Three ML Types Table:**

| Type | Method | Use Cases |
|------|--------|-----------|
| Supervised | Learn from labeled examples | Spam detection, credit scoring |
| Unsupervised | Find hidden structure | Customer segmentation, anomaly detection |
| Reinforcement | Learn by trial and reward | Game playing, robotics, trading |

**Key Insight:** "The algorithm that learns from data will always beat the algorithm coded by hand."
**Speaker Notes:** The breakthrough was simple. Stop writing rules. Show the machine examples. Let it find the patterns. Give a system ten thousand photos of cats and dogs. It learns to tell them apart. No one writes a rule for "whiskers." The algorithm discovers what matters.

**Headline:** The Transformer: The Single Architecture Powering the AI Revolution

### Slide 28: Multimodal AI
**Headline:** Multimodal: One System Working Across Multiple Data Types
**Visual:** Central AI node connected to: Text, Images, Audio, Video, Documents, Tables, Code, Sensor Data
**Four-Step Process:**
1. **Convert** — Each modality → embeddings
2. **Align** — Shared semantic space
3. **Fuse** — Cross-attention combines information
4. **Generate** — Classify, retrieve, generate, or act
**Key Insight:** "Most enterprise work is not pure text. Invoices include layout. Contracts include exhibits. Support cases include screenshots. The more modalities a system uses together, the closer it gets to understanding actual work."
**Speaker Notes:** Multimodal does not simply mean "has an image feature." It means the model can understand or generate across multiple kinds of data in one connected system. Most enterprise work is multimodal — invoices include layout and tables, contracts include clauses and exhibits, support cases include screenshots and logs. This is where AI becomes operationally meaningful.

### Slide 29: Mixture of Experts (MoE)
**Headline:** Mixture of Experts: Internal Specialization at Scale
**Visual:** Flow diagram: Input Token → Router → Expert 1/2/3/4 (only 2 activated) → Output
**Comparison Table:**

| Dimension | Multimodal | Mixture of Experts |
|-----------|-----------|-------------------|
| Addresses | Breadth of data types | Internal specialization |
| Primary benefit | Understanding real-world tasks | Scaling efficiently |
| Analogy | Company works with many materials | Company routes to right specialist |

**Speaker Notes:** Mixture of Experts is a model design where multiple specialized sub-networks exist inside one model. A router decides which expert handles each token. Only some experts activate — making the model large in capacity but efficient in computation. Multimodal is about what the model can see. MoE is about how the model thinks. Both are necessary.

---

### Slide 30: How LLMs Are Trained — The Three-Stage Process
**Headline:** From Raw Text to Aligned Intelligence — Three Training Stages
**Visual:** Three-stage pipeline:
1. **Pre-Training** — 300B tokens, self-supervised, next-word prediction. Learns language patterns.
2. **Supervised Fine-Tuning** — ~13,000 curated examples. Human experts write high-quality demonstrations.
3. **RLHF** — ~33,000 comparisons. Humans rank outputs. Reward model learns preferences.
**HHH Criteria:** Helpful, Honest, Harmless
**Speaker Notes:** An LLM is trained in three stages. First, it reads vast amounts of text and learns to predict the next word. Second, human experts write high-quality examples to teach instruction-following. Third, humans rank model outputs to align behavior with human preferences. The alignment objectives are Helpful, Honest, and Harmless — the HHH criteria.

### Slide 34: Vectorization — Turning Meaning into Mathematics
**Headline:** Vectorization Turns the Messy World into Mathematics
**Visual:** Illustration showing: Text document → Embedding model → Vector [0.23, -0.87, 0.45, ...] → Semantic space where similar concepts cluster

### Slide 35: RAG — Grounding AI in Enterprise Data
**Headline:** RAG: The Bridge Between General Intelligence and Your Specific Knowledge
**Visual:** Four-step RAG pipeline:

2. **Store** — Embeddings stored in vector database
3. **Retrieve** — System finds relevant chunks by meaning, not keywords

**Key Insight:** "RAG solves hallucinations by grounding answers in actual enterprise data. It accesses private knowledge LLMs cannot know on their own. And it keeps information current without retraining the model."
**Speaker Notes:** RAG is the most important enterprise AI architecture today. It solves the hallucination problem by grounding every answer in real data. It accesses private knowledge that LLMs cannot know on their own. And it keeps information current without retraining the model. RAG is the fuel that lets LLMs see the whole company, not just the public internet.
